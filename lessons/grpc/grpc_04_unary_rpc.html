Lesson 04 · Unary RPC 實戰

第一支 gRPC 服務:Unary RPC

從 proto 寫到 server、client 全部 hand on。本課使用 grpc-go v1.80.0 的新 API:grpc.NewClient() 取代已 deprecated 的 grpc.Dial()。

02 — 定義 proto

Step 1 — 寫 proto/greeter.proto

proto/greeter.proto 
syntax = "proto3";

package greeter.v1;

option go_package = "example.com/greeter/proto;greeterpb";

service Greeter {
  // Unary RPC:1 req → 1 resp
  rpc SayHello (HelloRequest) returns (HelloReply);
}

message HelloRequest {
  string name = 1;
}

message HelloReply {
  string message = 1;
}
03 — 生成 stub

Step 2 — 執行 protoc 與裝套件

bash 
# 在 greeter/ 根目錄
$ go mod init example.com/greeter

# 加入 grpc-go 與 protobuf runtime
$ go get google.golang.org/grpc@latest
$ go get google.golang.org/protobuf@latest

# 生成 stub
$ protoc \
    --go_out=. --go_opt=paths=source_relative \
    --go-grpc_out=. --go-grpc_opt=paths=source_relative \
    proto/greeter.proto

生成檔給了什麼

greeter_grpc.pb.go 會包含:

  • GreeterServer — 你要實作的 interface
  • GreeterClient — client 呼叫用的 interface
  • RegisterGreeterServer(s *grpc.Server, srv GreeterServer) — server 註冊
  • NewGreeterClient(cc *grpc.ClientConn) GreeterClient — client 建構
  • UnimplementedGreeterServer — 一定要 embed 進你的 server struct(處理 forward compatibility)
04 — 寫 Server

Step 3 — server/main.go

go 
package main

import (
    "context"
    "log"
    "net"

    "google.golang.org/grpc"
    pb "example.com/greeter/proto"
)

// 1. 定義 server struct,必須 embed UnimplementedGreeterServer
// 這個 embedding 提供 forward-compatible 的預設實作 —
// 將來 proto 加新 method 時,你的 server 不會編不過。
type server struct {
    pb.UnimplementedGreeterServer
}

// 2. 實作 SayHello — handler 簽名固定:
//    (ctx context.Context, req *PB) (*PB, error)
func (s *server) SayHello(ctx context.Context, req *pb.HelloRequest) (*pb.HelloReply, error) {
    log.Printf("received: %v", req.GetName())
    return &pb.HelloReply{
        Message: "Hello, " + req.GetName(),
    }, nil
}

func main() {
    // 3. 開 TCP listener
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatalf("failed to listen: %v", err)
    }

    // 4. 建 gRPC server,註冊我們的 implementation
    s := grpc.NewServer()
    pb.RegisterGreeterServer(s, &server{})

    log.Printf("server listening on %v", lis.Addr())

    // 5. Serve 會 block,直到 server 被 stop
    if err := s.Serve(lis); err != nil {
        log.Fatalf("failed to serve: %v", err)
    }
}
!
務必 embed pb.UnimplementedGreeterServer 這是 protoc-gen-go-grpc 強制的(你可以用 require_unimplemented_servers=false 關掉但不建議)。 它做兩件事:(1) 為將來新增 method 提供「Unimplemented 預設回傳」,讓你的 server 升級 proto 時不會編不過;(2) 強迫你寫 forward-compatible 的 service 實作。
05 — 寫 Client

Step 4 — client/main.go

go 
package main

import (
    "context"
    "log"
    "os"
    "time"

    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
    pb "example.com/greeter/proto"
)

func main() {
    name := "world"
    if len(os.Args) > 1 {
        name = os.Args[1]
    }

    // 1. 建立連線(注意:NewClient 是 lazy 的,這裡不會真的 dial)
    conn, err := grpc.NewClient(
        "localhost:50051",
        grpc.WithTransportCredentials(insecure.NewCredentials()),
    )
    if err != nil {
        log.Fatalf("failed to create client: %v", err)
    }
    defer conn.Close()

    // 2. 從生成檔拿到 client stub
    client := pb.NewGreeterClient(conn)

    // 3. 設一個 1 秒 deadline 的 context
    ctx, cancel := context.WithTimeout(context.Background(), time.Second)
    defer cancel()

    // 4. 真正的 RPC call(連線會在這裡實際建立)
    resp, err := client.SayHello(ctx, &pb.HelloRequest{Name: name})
    if err != nil {
        log.Fatalf("could not greet: %v", err)
    }
    log.Printf("greeting: %s", resp.GetMessage())
}
i
insecure.NewCredentials() 是什麼 告訴 gRPC「我這條連線不加密」 — 純明文 HTTP/2(h2c)。只在 localhost 開發、或在 service mesh 內部已有 mTLS 的情況下用。production 對外連線一定要 TLS,第 9 課會教。

07 — 舊 API vs 新 API

為什麼網路上很多教學 client 寫法不一樣?

因為 grpc-go v1.63.0(2024 初)把 client API 換掉了。如果你看到 grpc.Dial(...) 或 grpc.WithInsecure(),那是舊 API,已經 deprecated。

✗舊 API (deprecated)
conn, err := grpc.Dial(
    "localhost:50051",
    grpc.WithInsecure(),
    grpc.WithBlock(),
)

v1.63 後 deprecated,1.x 還能用但會警告。

✓新 API
conn, err := grpc.NewClient(
    "localhost:50051",
    grpc.WithTransportCredentials(
        insecure.NewCredentials(),
    ),
)

v1.63+ 推薦寫法。

三件你需要知道的差別

  1. NewClient 是 lazy — 呼叫時不會真的 dial,第一次 RPC 才會建連線。所以這幾個 DialOption 會被 NewClient 忽略:WithBlock()、WithTimeout()、WithReturnConnectionError()、FailOnNonTempDialError()。
  2. 預設 name resolver 從 passthrough 改為 dns — 寫 "foo.com:80" 時 NewClient 會做 DNS 解析、支援多個 backend address。
  3. WithInsecure() 換成 WithTransportCredentials(insecure.NewCredentials()) — 老寫法 import 簡單但無法擴充,新寫法統一用 credentials 套件介面。
i
新專案一律用 NewClient 舊專案維護時看到 Dial 不用急著換(1.x 還支援),但寫新 code 時請用新 API。本系列後續所有 client 範例全部用 NewClient。
