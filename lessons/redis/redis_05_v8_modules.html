01 — Stack 併回

Redis Stack 是什麼，為什麼又消失了

2022 年 Redis 公司推出 Redis Stack：把 RediSearch、RedisJSON、RedisTimeSeries、RedisBloom 四個模組打包成一個發行版，方便一鍵安裝。但這帶來雙頭管理：OSS 與 Stack 兩條版本線、模組要單獨更新。

Redis 8（2025 年 GA）把 Stack 全部併回 Open Source：

  • Redis Search：全文 / 向量 / 數值範圍 / 標籤索引
  • Redis JSON：JSON 文件儲存 + JSONPath 查詢
  • Redis TimeSeries：時序資料 + downsampling + 聚合
  • Redis Bloom：Bloom / Cuckoo / Top-K / Count-Min sketch / t-Digest

意思是：裝 Redis 8 即內建這些功能，不用再 FT.MODULE LOAD ...。Redis Stack 6.2 / 7.2 / 7.4 已於 2025-09-15 停止支援。

i
對你影響 以前要用 RediSearch / RedisJSON 的人，常被困在「我的 Redis 是 Stack 還是 OSS、能不能用 X」的版本焦慮。Redis 8.0+ 統一一條線：裝 Redis 8 就有全部。docker image redis:8.6 已含。
02 — Vector Sets

Vector Sets：原生向量資料結構

8.0 beta 引入 Vector Set（簡寫 VSET），是一個專為向量相似度設計的新資料結構（不是模組指令）。8.6 達到 GA 級效能：

  • vs 8.4，向量寫入快 43%、查詢快 58%（同硬體）
  • 支援多種 quantization（int8、binary）省記憶體
  • HNSW 演算法（跟 Cassandra Vector Search 同家族）
redis-cli 
# 加入向量到 vector set
VADD docs VALUES 3 0.9 0.1 0.0 "doc:1"
VADD docs VALUES 3 0.85 0.05 0.1 "doc:2"
VADD docs VALUES 3 0.1 0.9 0.0 "doc:3"

# 查詢:取最相似的前 2 筆
VSIM docs VALUES 3 0.88 0.08 0.05 COUNT 2
1) "doc:1"     # 距離最近
2) "doc:2"

# 看 set 大小
VCARD docs
(integer) 3

Vector Sets vs RediSearch 的 vector index

Redis 提供兩個向量介面：

  • Vector Sets (VSET / VADD / VSIM ...)：原生資料結構，輕量、純向量、API 簡潔。適合不需要 metadata filter 的場景。
  • RediSearch FT.SEARCH 的 VECTOR field：包在 search index 內，可結合 metadata（標籤、數值範圍）做混合搜尋。複雜場景用這個。
03 — Search

Redis Search：全文 + 向量 + 結構化

Redis Search 用 FT.CREATE 建索引，可以一次涵蓋多種欄位型態：

redis-cli 
# 1. 建一個 hash 索引,涵蓋全文 + 標籤 + 數值 + 向量
FT.CREATE idx:news ON HASH PREFIX 1 "news:" SCHEMA \
  title TEXT WEIGHT 5.0 \
  category TAG \
  views NUMERIC SORTABLE \
  emb VECTOR HNSW 6 TYPE FLOAT32 DIM 3 DISTANCE_METRIC COSINE

# 2. 灌資料
HSET news:1 title "TSLA up"     category "stock" views 1200 emb "\x66\x66..."
HSET news:2 title "AAPL earnings" category "stock" views 800  emb "\x..."
HSET news:3 title "Inflation"    category "macro" views 3000 emb "\x..."

# 3. 全文搜
FT.SEARCH idx:news "TSLA"

# 4. 標籤 + 數值範圍
FT.SEARCH idx:news "@category:{stock} @views:[500 +inf]"

# 5. 向量 KNN
FT.SEARCH idx:news "*=>[KNN 2 @emb $q]" PARAMS 2 q "\x..."

# 6. 混合:在 stock 類別內做 KNN
FT.SEARCH idx:news "@category:{stock} =>[KNN 2 @emb $q]" PARAMS 2 q "\x..."

RediSearch 的甜蜜點是「不大不小」的搜尋場景：資料量幾百萬筆、混合過濾、要求低延遲、團隊已經有 Redis。比起拉一個 Elasticsearch 集群輕量很多。

04 — JSON

Redis JSON：第二類 native 結構

Redis JSON 提供 JSON 文件的原生儲存與 JSONPath 查詢：

redis-cli 
JSON.SET user:1 $ '{"name":"Jason","age":28,"orders":[{"id":1,"amt":100}]}'
OK

JSON.GET user:1 $.name
"[\"Jason\"]"

JSON.GET user:1 '$.orders[*].amt'
"[100]"

# 修改其中一個欄位 (不用整份重寫)
JSON.SET user:1 $.age 29

# 在陣列尾加東西
JSON.ARRAPPEND user:1 $.orders '{"id":2,"amt":50}'

RediSearch 也支援 ON JSON：可以直接對 JSON 內的 JSONPath 路徑做索引。寫起來像 MongoDB 但延遲是 Redis 等級。

05 — 8.6 亮點

8.6 帶來的具體改進

2026/03 公告。Redis 8 系列維持「每隔幾個月一個 minor 版本」的快節奏。8.6 重點：

  • Vector Sets 大幅加速：vs 8.4，VADD +43%、VSIM +58%；針對 Intel / AMD / ARM 用 SIMD 與 popcount 內建函式優化二進位向量距離計算。
  • 30+ 整體效能改進：跨指令的小優化累積。
  • Streams 增強：consumer group 操作更高效；新增更細的 entry 過濾。
  • 新 Eviction Policies：除了傳統 LRU/LFU/random，新增更聰明的策略（例如針對 hash field 級別的 TTL 與 eviction）。
  • Hash Field TTL（8.0 引入）：個別 hash 欄位可有自己的過期時間，HEXPIRE / HEXPIREAT / HPERSIST。
redis-cli 
# Hash Field TTL (8.0+)
HSET session:abc user_id 100 last_seen 1745912345
HEXPIRE session:abc 1800 FIELDS 1 last_seen
# last_seen 欄位 30 分鐘後過期,user_id 不過期
06 — 取捨

什麼時候用 Redis 8 取代 ES / Pinecone / pgvector？

場景建議
RAG / 向量搜尋 (< 1000 萬筆)，team 已經有 Redis Redis Vector Sets / RediSearch
向量規模 > 1 億筆，需要進階調參 Pinecone / Milvus / Weaviate
全文搜尋為主、要 BM25 / 同義詞 / 多語言分析器 Elasticsearch
已用 PostgreSQL，向量 + 結構化資料同庫 pgvector
輕量 JSON 文件儲存 + 低延遲查 Redis JSON（取代 MongoDB 的小場景）

Redis 的賣點永遠是「低延遲 + 極簡部署 + 已經在你架構裡」。當你的場景不需要 search engine 等級的全文功能、向量規模中等，Redis 8 是多開一個元件 vs 用現有 Redis 的取捨。

07 — 動手玩

跑 Vector Set + RediSearch

bash 
# 確認用的是 Redis 8.6 (8.0+ 含模組)
$ docker run --rm -p 6379:6379 redis:8.6

$ docker exec -it $(docker ps -q --filter ancestor=redis:8.6) redis-cli
> INFO server | grep redis_version
redis_version:8.6.x

# 1. Vector Set 玩法
> VADD mySet VALUES 3 0.9 0.1 0.0 "doc:tsla"
> VADD mySet VALUES 3 0.85 0.1 0.05 "doc:aapl"
> VADD mySet VALUES 3 0.1 0.9 0.0 "doc:macro"
> VSIM mySet VALUES 3 0.88 0.08 0.05 COUNT 2
1) "doc:tsla"
2) "doc:aapl"

# 2. RediSearch + JSON
> JSON.SET user:1 $ '{"name":"Jason","city":"Taipei","age":28}'
> JSON.SET user:2 $ '{"name":"Alex","city":"Tokyo","age":35}'

> FT.CREATE idx:user ON JSON PREFIX 1 "user:" SCHEMA \
    $.name AS name TEXT \
    $.city AS city TAG \
    $.age  AS age NUMERIC SORTABLE

> FT.SEARCH idx:user "@city:{Taipei}"
> FT.SEARCH idx:user "@age:[30 +inf]"
